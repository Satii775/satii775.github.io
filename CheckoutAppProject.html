Accessible Checkout App

AR-guided navigation system for BSU GIMM equipment checkout, designed for accessibility.

Swift ARKit RealityKit VisionKit SwiftUI

Project Details

Role
AR Developer
Timeline
Feb 2025 - May 2025
Team
3 members

Overview

The Accessible Checkout App is an AR-powered navigation system designed for the Boise State University GIMM department's equipment checkout process. Built with a team of 3, this app specifically addresses the needs of students with Autism by providing clear visual guidance, audio instructions, and a structured checkout flow.

Using ARKit and image recognition, the app guides users through the physical space from the front desk to check-in and check-out stations, making the equipment borrowing process more accessible and less overwhelming.

Goals & Constraints

  • Accessibility First: Design specifically for users with Autism, minimizing sensory overload while providing clear guidance.
  • AR Navigation: Implement image anchor-based wayfinding with visual path indicators between stations.
  • Equipment Variety: Support checkout of various tech equipment including VR headsets, cameras, and microphones.
  • Constraint: Work within the existing GIMM checkout infrastructure and physical space layout.
  • User Testing: Validate accessibility features with actual users and iterate based on feedback.

Process

The development process focused on creating an intuitive AR experience that reduces cognitive load. We implemented a multi-anchor system using ARKit's image recognition capabilities, where each physical location (front desk, check-in QR station, check-out QR station) has a dedicated AR anchor.

Key Technical Implementations:

  • Image Anchor System: Used ARWorldTrackingConfiguration with custom reference images to create persistent anchors at key locations.
  • Sequential Visibility: Implemented progressive disclosure where anchors become visible only when needed, preventing overwhelming information.
  • Proximity Menus: Context-sensitive menus appear when users approach anchors, providing clear next-step options.
  • Path Visualization: Created visual paths between anchors using RealityKit to guide users through the space.
  • Audio Guidance: Integrated text-to-speech for accessibility, announcing each step in the process.
  • State Management: Built a comprehensive checkout flow state machine to handle check-in and check-out processes.

Accessibility Considerations:

  • Clear, high-contrast visual indicators
  • Consistent UI patterns throughout the experience
  • Predictable navigation flow with minimal surprises
  • Audio feedback for all major interactions
  • Option to pause and review instructions at any point

Outcomes

  • Successful AR Navigation: Implemented a working 4-anchor system with image recognition and path visualization.
  • Accessibility Features: Integrated audio guidance, proximity-based menus, and clear visual indicators specifically designed for users with Autism.
  • Equipment Management: Built support for checking out various tech equipment including VR headsets, microphones, and cameras.
  • Persistence System: Implemented AR world map saving/loading to maintain anchor positions across sessions.
  • Team Collaboration: Successfully coordinated with 2 teammates to deliver a functional prototype within the semester timeline.

Reflections

What Worked Well: The image anchor approach proved very effective for indoor navigation. The sequential visibility system helped reduce cognitive overload by showing users only relevant information for their current step. Audio guidance was particularly successful in providing reassurance without requiring constant visual attention.

Challenges: Managing AR session persistence was more complex than anticipated. We needed to implement robust error handling for cases where image anchors weren't immediately detected. Balancing the information displayed in proximity menus while keeping them accessible required multiple iterations.

Future Improvements: With more time, I would add haptic feedback for approaching anchors, implement a practice mode for first-time users, and expand the system to handle equipment return documentation with photo capture. User testing with individuals with Autism would provide invaluable feedback for refining the accessibility features.

Key Learnings: This project deepened my understanding of ARKit's capabilities, particularly around world tracking and image recognition. I learned the importance of designing for specific user needs rather than general audiences, and how small UX details can significantly impact accessibility. Working in a team of 3 on an AR project taught me valuable lessons about coordinating development when testing requires physical space and specific conditions.
